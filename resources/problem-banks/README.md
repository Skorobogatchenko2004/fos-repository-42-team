# Банки задач и кейсов

Практические кейсы дисциплины. Для каждого кейса указываются проверяемый результат обучения, сложность, входные данные, подход к проверке и варианты параметризации.

| Кейс | Проверяемый результат | Входные данные | Связанные КИМ | Дата проверки |
|---|---|---|---|---|
| Прогнозирование спроса для сети розничных магазинов | РО-1 … РО-5 | описание бизнес-процессов, данные продаж, ИТ-окружение (ERP, WMS, BI) | КИМ-1.5, КИМ-2.5, КИМ-3.4, КИМ-4.1, Проект | 17.07.2026 |

## Сквозной кейс

Кейс прогнозирования спроса является сквозным: его артефакты передаются между модулями (Feasibility Study → требования → архитектура → верификация → проект). Параметризация вариантов для анализа влияния изменения (развёртывание on-premise, изменение горизонта прогноза, добавление франчайзи) относится к [материалам преподавателя](../../methodical-guidelines/teachers-assessment/README.md).

Проектная работа может выполняться на кейсе от индустриального партнёра при сохранении требований к составу артефактов.
