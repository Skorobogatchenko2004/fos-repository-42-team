# Рабочая программа дисциплины «Системная инженерия для систем искусственного интеллекта»

## 1. Общие сведения

| Поле | Значение |
|---|---|
| Название дисциплины | Системная инженерия для систем искусственного интеллекта |
| Образовательная программа | Преподавание современного ИИ на топ-уровне: системное видение |
| Уровень образования | Бакалавриат, 5 семестр |
| Объём | 108 академических часов / 3 зачётные единицы |
| Формат обучения | Смешанный |
| Форма промежуточной аттестации | Зачёт с защитой проектной работы |

Дисциплина является интегрирующей дисциплиной профессионального цикла. Она формирует системное мышление и практические навыки управления жизненным циклом промышленных систем искусственного интеллекта и ориентирована на профессиональные компетенции ролей AI-PM и ML Engineer, зафиксированные в компетентностно-ролевой модели в области искусственного интеллекта (КРМ версии 3.0).

## 2. Цель и задачи

**Цель.** Сформировать системное мышление, навыки анализа сложных инженерных задач, разработки проектной документации и применения методов системной инженерии для программно-аппаратных комплексов, а также компетенции в проектировании, разработке и управлении жизненным циклом сложных систем с интеграцией компонентов искусственного интеллекта с учётом требований надёжности, безопасности и этичности.

**Задачи.**

1. Освоить методы анализа бизнес-проблем и оценки целесообразности применения искусственного интеллекта.
2. Научиться разрабатывать техническую документацию (техническое задание, требования, спецификации) для систем искусственного интеллекта.
3. Изучить стандарты проектирования и инструменты системного моделирования.
4. Освоить методологии системной инженерии применительно к системам искусственного интеллекта.
5. Сформировать навыки управления требованиями и архитектурного проектирования систем с компонентами искусственного интеллекта.
6. Развить способность оценивать риски, связанные с недетерминированным поведением моделей искусственного интеллекта.
7. Приобрести практические навыки модельно-ориентированной системной инженерии (MBSE) для систем искусственного интеллекта.
8. Сформировать компетенции в области управления качеством данных, а также правовых и этических аспектов внедрения искусственного интеллекта.

## 3. Пререквизиты

Для освоения дисциплины обучающийся должен обладать следующими результатами предшествующего обучения.

| Дисциплина | Компетенция | Индикатор | Уровень |
|---|---|---|:---:|
| Программирование | PL-1. Способен применять язык программирования Python для решения задач в области ИИ | PL-1.1 | С |
| Управление данными | BD-3. Способен организовывать хранение данных, выбирая адекватные технологические решения | BD-3.1 | Б |
| Основы искусственного интеллекта | ML-2. Способен применять фундаментальные принципы и методы машинного обучения | ML-2.1 | Б |

## 4. Планируемые результаты обучения

По завершении дисциплины обучающийся способен:

- **РО-1.** Проводить анализ бизнес-проблемы и оценивать целесообразность применения искусственного интеллекта, формализуя бизнес-цели и требования заинтересованных сторон (LC-1.1).
- **РО-2.** Разрабатывать требования и техническую документацию на систему искусственного интеллекта, обеспечивая их сквозную трассируемость и соответствие действующим стандартам (LC-1.2, LC-1.3, ОПК-4.1).
- **РО-3.** Проектировать архитектуру системы искусственного интеллекта с обоснованием выбора технологического стека и архитектурных решений (LC-3.1).
- **РО-4.** Планировать и контролировать процессы жизненного цикла продукта на основе искусственного интеллекта, выбирая методологию под ограничения задачи (LC-4.1).
- **РО-5.** Организовывать верификацию, валидацию и сопровождение систем искусственного интеллекта, оценивая качество данных и моделей и принимая решение о вводе в эксплуатацию (LC-3.1, LC-4.1).

## 7. Система оценки результатов обучения

Оценивание построено по балльно-рейтинговой системе. Текущий контроль проводится по каждому модулю в форме защиты лабораторных работ, практических заданий и деловых игр; промежуточная аттестация — зачёт с защитой проектной работы.

Каждый результат обучения подтверждается контрольно-измерительными материалами соответствующих модулей и проектной работой. Соответствие результатов обучения, индикаторов, форм контроля и контрольно-измерительных материалов приведено в модели измерения (раздел 2 README репозитория). Правила начисления баллов, пороги и перевод в итоговую оценку приведены в системе итогового оценивания [`docs/grading.md`](grading.md).

Использование генеративного искусственного интеллекта при выполнении заданий разрешено и подлежит обязательному декларированию. Правила приведены в методических указаниях обучающимся [`methodical-guidelines/students/README.md`](../methodical-guidelines/students/README.md).
