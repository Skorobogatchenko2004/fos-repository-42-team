# Программа и методика испытаний (шаблон)

Разрабатывается до запуска проверок. Критерии «годен / не годен» задаются числовыми значениями заранее.

## 1. Объект и цели испытаний

- Объект: модель прогнозирования спроса (версия, источник).
- Цель: проверка пригодности модели к вводу в промышленную эксплуатацию.
- Границы применимости модели.

## 2. Baseline

Базовое решение для сравнения (например, скользящее среднее за 28 дней) и его показатели на тех же данных.

## 3. Перечень проверок

| № | Проверка | Метрика | Критерий «годен» | Данные проверки |
|---|---|---|---|---|
| 1 | Качество данных |  |  |  |
| 2 | Утечка признаков |  | утечка отсутствует |  |
| 3 | Качество модели против baseline |  |  | хронологически отложенная выборка |
| 4 | Качество по разрезам |  |  | категории, форматы, регионы |
| 5 | Дрейф |  |  | обучающая выборка и свежий срез |
| 6 | Устойчивость |  |  | данные с пропусками и шумом |

## 4. Требования к данным испытаний

Обоснование нарезки данных: почему выбрано хронологическое, а не случайное разбиение.

## 5. Порядок фиксации результатов

Формат выгрузки результатов проверок и место их хранения в репозитории (каталог `reports/`).
