# Спецификация данных и модели для КИМ-4.1

Материал для преподавателя. Датасет и модель с намеренным дефектом готовятся преподавателем на основе открытых данных; в публичный репозиторий закладываемый дефект и его описание не помещаются. Настоящий файл задаёт требования к подготовке.

## 1. Исходные данные

За основу берётся открытый набор данных розничных продаж, пригодный для задачи прогнозирования спроса (день, товар, магазин, продажи, признак промоакции). Требования к набору: не менее двух полных годовых циклов, наличие товарных категорий и нескольких форматов торговых точек.

## 2. Три среза данных

| Срез | Назначение |
|---|---|
| Обучающая выборка | обучение модели |
| Тестовая выборка | хронологически отложенная, для оценки качества |
| Свежий срез | последний период, для проверки дрейфа |

Разбиение выполняется по времени. Свежий срез отражает распределение, отличное от обучающего (например, содержит период с промоакциями или новую категорию).

## 3. Модель

Обучается модель прогнозирования спроса и сохраняется вместе с отчётом команды разработки. Отчёт содержит достойные агрегированные метрики на тестовой выборке и рекомендацию к внедрению; отчёт не упоминает закладываемый дефект.

## 4. Виды закладываемого дефекта

Преподаватель выбирает один или два дефекта и готовит несколько вариантов для разных обучающихся:

1. **Утечка целевой переменной** — в признаки включено поле, недоступное на момент прогноза (например, фактическая выручка дня).
2. **Деградация во времени** — тестовая выборка нарезана случайно, а не по времени; на хронологически отложенном срезе качество падает ниже baseline.
3. **Перекос по подгруппам** — средняя метрика в норме, но по отдельной категории или формату качество хуже baseline.
4. **Смещение обучающих данных** — данные собраны в период без промоакций, при этом в эксплуатации промо составляют значимую долю оборота.
5. **Качество ниже baseline** — модель уступает скользящему среднему, но baseline в отчёте не приведён.

## 5. Материалы преподавателя (закрытые)

Описание заложенного дефекта, скрипт его внесения и эталонное решение хранятся вне публичного репозитория. Варианты обновляются ежегодно.
